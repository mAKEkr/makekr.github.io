Github Pages로 Ghost 이용하기 #2 - 설정하기 및 발행하기

12 August 2016 
여기서부터 Bash Shell과 CMD(PowerShell)을 혼합하여 사용할것이다. 만일 이해가 안간다고 생각되면 즉시 중단하고 게시물부터 차근차근 다시한번 꼼꼼히 읽어가는걸 추천한다.

Ghost 기본 설정

Ghost의 설정파일은 기본 디렉터리의 config.js 파일이다. config.js파일 안을 들여보면 다음과 같은 코드들이 보이는데, 해당 내용만 수정해주면 된다.

        url: 'http://127.0.0.1:2368'

여기에 있는 127.0.0.1:2368을 자신의 도메인(ex: http://makekr.github.io)으로 변경한다. 커스텀도메인의 경우 추가설정이 필요하니 아래에 있는 Github Pages로 발행하기 부분을 참고하길 바란다.

Buster 설정

이제 Buster를 설정해보자.

* BASH Shell에서 작업하도록 하자 *

  1. ghost가 설치된 폴더로 이동한다.
  2. buster setup을 입력한다.
  3. Enter the Github repository URL: 이라고 메세지가 뜬다. github저장소 주소(본인같은경우 https://github.com/mAKEkr/makekr.github.io) 를 입력한다.

커스텀 도메인 설정

Github Pages는 기본적으로 발행해주는 github.io 도메인 대신에 개인이 구매한 도메인을 사용할 수 있도록 만들어준다.

사용방법은 다음과같다.

  1. 원하는 도메인의 A클래스를 192.30.252.153 IP로 설정한다.
  2. 도메인이 제대로 등록됨을 기다린다(최소 10분, 최대 6시간 이상 걸린다. DNS서버에 따라 다르다)
  3. buster add-domain 도메인이름을 입력해준다.(ex: ake.kr이면 buster add-domain ake.kr)

Github Pages로 발행하기

이제 설정이 완료되었다. 제대로 Github에 올라가는지 확인하기 위하여 한번 확인을 해볼 필요가 있다.

  1. 설정이 완료되면 제대로 작동하는지 확인하기 위해 윈도우에서 Ghost를 실행시킨다.
  2. Ghost 실행이 완료되면 buster generate를 입력한다
  3. buster deploy를 입력한 뒤에 Github아이디와 Github 비밀번호를 입력하여 Deploy가 제대로 되는지 확인한다.

Deploy가 완료되면 이런식으로 메세지가 뜬다.

테마 파일 복사하기

Ghost로 github pages를 발행하게되면 CSS가 없는 HTML만 있는 화면이 나온다. "분명 튜토리얼을 따라 잘 했는데!" 라면서. 무척 당황스러울것이다. 나도 그랬다.

원인은 다음과같다.

테마에 해당되는 CSS/JS가 복사가 안된다!

이에 개의치않고, 우리는 수동으로 복사를 해줘야한다..

  1. (Ghost 설치폴더)/content/themes/(테마 폴더)/assets/에 들어간다.
  2. (Ghost 설치폴더)/static/assets/에 들어간다.
  3. 1번의 폴더내용을 2번에 복사한다.
  4. ghost deploy로 Github Pages에 올려준다.

끝이다.

그리고 이 테마복사가 끝나면 유의해야할것이, assets폴더에 있는 내용들을 아무리 수정해도 우리가 github pages에 올리는 내용들엔 안바뀐다는 것이다. 항시 assets 폴더를 복사해야함을 명심하자.

오류가 뜰경우 생각해볼것

  • Git을 설치하지 않았는가?
  • Buster를 제대로 설치했는가?
  • Bash Shell에서 Ghost를 실행시킨것은 아닌가? (다시금 말하지만 현재 윈도우의 Bash Shell은 불안정하다)
  • git 저장소의 주소를 제대로 입력했는가?
  • git 설정은 제대로 되어있는가?

CSS가 적용되지않는 경우는 다음 3번째장에서 알아보도록 한다.
