Github Pages로 Ghost 이용하기 #Intro - 황혼부터 삽질까지

08 August 2016 on Ghost, Tutorial

블로그를 세번이나 갈아엎었다. 내 홈페이지에 자주 방문할 사람이 있을까 싶지만, "이새끼는 뭐하는데 블로그를 자주 갈아엎어?" 라면서 물어볼때를 대비하여 댈만한 이유들은 참 많지만 핵심적인 이유들을 꼽자면

  1. XE를 더이상 안쓴다
  2. 워드프레스가 더이상 마음에 안든다
  3. 자원도 얼마 안들어가는 블로그인데 코노하에 10달러씩 꼬박꼬박 바치고있는게 배가 아프다

이게 핵심적인 요소들이다(3번은 좀 이상하긴 한데, 그렇다고 하자). 그래서 생각해놓은것이 많은 개발자들이 사용하던 정적페이지 호스팅 블로그. 즉 Jekyll을 이용한 블로그! 이것이 가장 마음에 들었다.

근데 Jekyll을 쓰자니 이런것들이 걸렸다

  • Ruby의 버전을 두번이나 갈아엎어야했고, rvm은 완벽히 작동하지 않았다
  • Jekyll의 사용을 하기위해서는 Ruby/Node/Python 즉 세개의 언어가 필요했다.
  • 메뉴얼이 정말 불친절했다.

그래서 생각해두고있었던것이 바로 Ghost, 이 Ghost는 상당히 잘 만들어진 블로그 플랫폼이였고, 오픈소스형태의 사용방식도 있어서 주저말고 Ghost를 사용하기로 선택했다. 무엇보다 대시보드가 깔끔하여 마크다운으로 게시물을 작성하기에도 상당히 편리했고.

환경

  • Windows 10 Pro x64 레드스톤 빌드(이른바 1주년 업데이트)
  • Windows Bash(Ubuntu base)

시행착오

시행착오는 솔직히 한탄에 가깝다. 그러니까 불필요하다 싶으면 설치과정으로 넘어가시는걸 적극권장 한다.

솔직히 설치 처음부터 상당히 애를먹었다. 아무런 문서도 없고, 아무런 사람들의 이야기도 없이 설치한다는게 워낙 고역이다...

멘붕의 현장 1. 노드는 4버전대(LTS)를 지원한다. 6버전대는 지원이 안된다...

물론 이것은 package.json을 뜯어고치면 되는 문제긴 하지만, 솔직히 여기서 맞부닺힐지 몰랐다. 그리고나서 두번째 멘붕이 시작된다.

멘붕의 현장 2. 윈도우 배쉬쉘에선 ghost를 돌릴때 uv_interface_addresses 오류로 인하여 정상작동하지 않았다.

이 문제도 '내가 Ghost를 잘못설치했나?'라고 생각했지만, 똑바로 설치한거였고 결국엔 window bash shell 문제임이 깃허브 이슈를 통해 확정되는 순간이였다. Issue

그 다음은 ghost를 정적페이지로 바꾸어주는 buster가 문제였다. Wow... Buster 패키지를 설치하려면 Python 설치 -> pip 설치(윈도우 패키지 안엔 포함되어있다) -> buster 설치인데, buster에서 사용하는 lxml 패키지때문에 막혔다.

일단 설치를 위한 멘붕과정은 끝났다. 그리고 또다시 멘붕할것이 남아있다... 그건 1장 설치에서 알아보자..
